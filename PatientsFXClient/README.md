# PatientsFXClient

JavaFX клиентское приложение для управления пациентами через REST API с использованием Spring RestTemplate.

## Описание

Это отдельное JavaFX приложение, которое взаимодействует с REST API сервера Patients через Spring RestTemplate. Приложение предоставляет графический интерфейс для:
- Просмотра списка пациентов в таблице
- Добавления новых пациентов
- Обновления существующих пациентов
- Удаления пациентов

## Требования

- Java 21 или выше
- Maven 3.6+
- Запущенный сервер Patients на `http://localhost:8080`

## Запуск

1. Убедитесь, что сервер Patients запущен и доступен на `http://localhost:8080`

2. Запустите приложение одним из способов:

   **С помощью Maven:**
   ```bash
   mvn clean javafx:run
   ```

   **Или скомпилируйте и запустите вручную:**
   ```bash
   mvn clean compile
   java --module-path <path-to-javafx> --add-modules javafx.controls,javafx.fxml -cp target/classes com.patientsfx.Main
   ```

## Использование

### Просмотр пациентов
- При запуске приложение автоматически загружает список пациентов из REST API
- Для обновления списка нажмите кнопку "Обновить список"

### Добавление пациента
1. Заполните форму:
   - **Имя** - обязательное поле
   - **Дата рождения** - обязательное поле (не может быть в будущем)
   - **Диагноз** - обязательное поле
2. Нажмите кнопку **"Добавить"**
3. Список автоматически обновится

### Обновление пациента
1. Выберите пациента в таблице (кликните на строку)
2. Данные пациента автоматически заполнят форму
3. Измените нужные поля
4. Нажмите кнопку **"Обновить"**
5. Список автоматически обновится

### Удаление пациента
1. Выберите пациента в таблице
2. Нажмите кнопку **"Удалить"**
3. Подтвердите удаление в диалоговом окне
4. Список автоматически обновится

### Очистка формы
- Нажмите кнопку **"Очистить"** для очистки формы и снятия выделения

## Технические детали

### Использование Spring RestTemplate
Проект использует Spring RestTemplate для взаимодействия с REST API, что обеспечивает:
- Автоматическую сериализацию/десериализацию JSON
- Поддержку LocalDate через Jackson JSR310 модуль
- Упрощенную обработку HTTP запросов
- Типобезопасность при работе с данными

### REST API Endpoints
- `GET /api/patients` - получение всех пациентов
- `GET /api/patients/{id}` - получение пациента по ID
- `POST /api/patients` - создание нового пациента
- `PUT /api/patients/{id}` - обновление пациента
- `DELETE /api/patients/{id}` - удаление пациента

## Настройка URL сервера

По умолчанию приложение подключается к `http://localhost:8080/api/patients`. 
Чтобы изменить URL, отредактируйте константу `BASE_URL` в классе `PatientApiService`.
